## 🚀 错误库使用流程

每次开发任务开始前：

```
1. 查阅本 index.md → 确认是否有与当前任务相关的历史错误
2. 查阅相关 ERR-XXX.md → 理解根因与预防措施
3. 在设计阶段主动规避 → 将预防措施加入设计文档 checklist
4. 完成代码后 → 由 Reviewer 再次校验预防措施是否落实
```

发现新错误时：

```
1. 立即创建新的 ERR-XXX.md（参考下方模板）
2. 填入错误描述、复现步骤、根本原因
3. 提出修复方案并与用户确认 → 进入 WORKFLOW.md 的修复流程
4. 修复验证后 → 写入"预防措施"与"关联原则"
5. 将新条目加入本 index.md 的分类索引
```
