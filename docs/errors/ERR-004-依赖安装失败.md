# [ERR-004] npm install 依赖安装失败 / 版本冲突

| 字段 | 内容 |
|------|------|
| **错误编号** | ERR-004 |
| **发现日期** | 2026-06-16 |
| **严重程度** | 🟡 中 |
| **涉及模块** | 构建工具链 |
| **涉及文件** | `package.json`、`package-lock.json`、`node_modules/` |
| **发现阶段** | 环境准备 / 首次 clone 后 |
| **状态** | ✅ 已解决 |

---

## 错误现象

执行 `npm install` 时失败：

- `ERESOLVE could not resolve` —— 依赖版本冲突
- `ETARGET package not available` —— 包名拼写错误或版本不存在
- `ENOENT cannot find module` —— 缺少 peerDependencies
- `ERESOLVE unmet peer dependency` —— 主包要求的 peer 依赖未安装

**预期行为**：`npm install` 成功完成，`node_modules/` 目录完整。  
**实际行为**：安装中断，缺少关键依赖，TypeScript / Metro 无法启动。

## 复现步骤

1. 全新 clone 仓库：`git clone <url>`
2. 进入目录：`cd inspiration-bartender`
3. 执行：`npm install`
4. 观察到上述任一类错误

---

## 根本原因分析

| 常见子类 | 说明 |
|---------|------|
| **版本锁冲突** | `package.json` 声明了宽松版本（`^` / `~`），但 lock 文件版本与最新 npm 源不兼容 |
| **Expo SDK 版本漂移** | `expo` 版本与其他 expo-* 包版本不一致 |
| **peerDependencies 未装** | 主包声明了 peer 依赖但未在 `package.json` 中显式声明 |
| **锁文件过期** | `package-lock.json` 与 `node_modules/` 不一致导致校验失败 |

深层原因：Ponytail 精简原则（原则一）未严格控制依赖引入数量；新引入包未经过审查。

---

## 解决方案

### 修复步骤

```bash
# 1. 清理后重装（推荐）
rm -rf node_modules package-lock.json
npm install

# 2. 指定 Expo SDK 版本（确保版本一致）
npx expo install --check
# 或手动修改 package.json 指定兼容版本后重装

# 3. 如果是 peer 依赖问题
npm install --legacy-peer-deps

# 4. 验证安装结果
npx expo start --offline   # 确认 Metro 能启动
npx tsc --noEmit           # 确认类型检查正常
```

### 灵感调酒师当前依赖版本约束

```
React Native: ~0.76.x          （与 expo ~52.x 匹配）
Expo SDK: ~52.x
Zustand: ^4.x
React Native Reanimated: ^3.x
expo-sqlite: ^15.x
```
