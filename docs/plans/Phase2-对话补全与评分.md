## 一、阶段目标

Phase 2 为灵感调酒师添加智能对话功能：
- ✅ 与灵感进行对话式交互
- ✅ AI 自动追问和补充建议
- ✅ 评估灵感完整度
- ✅ 提供改进建议
- ✅ 支持多种 LLM 提供商

| 状态映射 | 分数→种子/萌芽/生长/含苞/绽放 | ✅ |

### 2.4 功能详情

**调酒工作台**
- 顶部显示当前灵感的大杯子
- 中间显示对话历史（气泡样式）
- 用户输入区域
- 发送按钮
- 液体高度随对话进展动画上升

**评分系统**
- 自动评估灵感内容
- 四维评分展示
- 可视化显示分数
- 提供具体改进建议
- 评分结果影响杯子液体高度

**LLM调用**
- 支持三种主流提供商
- 自动构建对话上下文
- 错误处理和重试机制
- Token使用统计

## 三、新增架构

### 3.1 新增文件

```
src/
├── services/llm/
│   ├── config.ts         # LLM配置管理
│   ├── provider.ts        # LLM提供商实现
│   ├── index.ts          # 导出入口
│   └── prompts/
│       ├── brewing.ts    # 调酒对话提示词
│       └── scoring.ts    # 评分提示词
├── services/scoring/
│   ├── dimensions.ts     # 评分维度定义
│   ├── evaluator.ts      # 评分器实现
│   └── index.ts          # 导出入口
├── screens/BrewingScreen/
│   ├── index.tsx         # 调酒工作台主界面
│   ├── ChatBubble.tsx    # 对话气泡组件
│   └── ProgressIndicator.tsx # 进度指示器
└── store/
    └── brewingStore.ts   # 调酒状态管理
```

### 3.2 数据流

```
用户输入
    ↓
构建消息上下文（系统提示 + 历史消息）
    ↓
调用LLM API（OpenAI/Anthropic/Ollama）
    ↓
解析响应
    ↓
更新UI（显示回复）
    ↓
更新评分（重新计算完整度）
    ↓
触发动画（液体高度变化）
    ↓
持久化数据（保存到SQLite）
```
