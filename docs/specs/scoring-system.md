## 一、模块概述

### 1.1 功能定位
评分系统评估灵感的完整度，为用户提供：
- 四维评分：清晰度、丰富度、可行性、独特性
- 综合评分：加权计算总分（0-100）
- 状态映射：将分数映射到5个成长阶段
- 改进建议：为低分维度提供改进方向

### 1.2 文件结构

```
src/services/scoring/
├── dimensions.ts      # 评分维度定义和配置
├── evaluator.ts        # 评分器实现（核心）
└── index.ts           # 导出入口
```

## 二、评分维度设计

### 2.1 四维评分模型

| 维度 | 英文标识 | 权重 | 满分 | 评估重点 |
|------|---------|------|------|---------|
| **清晰度** | CLARITY | 25% | 100 | 表达是否清晰明确 |
| **丰富度** | RICHNESS | 30% | 100 | 内容是否多样丰富 |
| **可行性** | FEASIBILITY | 25% | 100 | 是否可执行、有方案 |
| **独特性** | UNIQUENESS | 20% | 100 | 是否独特有创新 |

**权重和为 1.0（100%）**

### 2.2 维度详细说明

#### 清晰度（CLARITY）- 权重 25%
**评估指标**：
- 灵感名称（最多25分）
  - 有明确名称（非"新灵感"）：25分
  - 使用默认名称：0分

- 文本描述长度（最多40分）
  - ≥100字符：40分
  - ≥50字符：30分
  - >0字符：15分
  - 无文本：0分

- 结构化内容（最多35分）
  - 每个结构化字段：10分
  - 最高35分

**目标**：确保灵感有清晰的表达和基本描述

#### 丰富度（RICHNESS）- 权重 30%
**评估指标**：
- 图片内容（最多20分）
  - 每张图片：10分，最高20分

- 语音内容（最多15分）
  - 有语音记录：15分

- 链接内容（最多15分）
  - 有参考链接：15分

- 头脑风暴卡片（最多30分）
  - 每张卡片：10分，最高30分

- 酿造日志（最多20分）
  - 每条记录：5分，最高20分

**目标**：鼓励多模态、多层次的内容表达

#### 可行性（FEASIBILITY）- 权重 25%
**评估指标**：
- 基本描述（最多20分）
  - 文本超过20字符：20分

- 头脑风暴卡片（最多40分）
  - 每张卡片：15分，最高40分

- 碰撞历史（最多30分）
  - 每次碰撞：10分，最高30分

- 执行计划（最多10分）
  - 有steps/plan/actions字段：10分

**目标**：评估灵感是否经过深思熟虑，是否可执行

#### 独特性（UNIQUENESS）- 权重 20%
**评估指标**：
- 基础分（20分）
  - 每个灵感都有一定独特性

- 碰撞历史（最多30分）
  - 每次碰撞：15分，最高30分

- 容器类型（15分）
  - flask/beaker/martini等创意类型：15分

- 头脑风暴卡片数量（最多20分）
  - ≥3张：20分
  - >0张：10分

- 多模态输入（最多15分）
  - ≥3种输入类型：15分
  - ≥2种输入类型：8分

**目标**：鼓励创新思维和跨领域连接

## 三、状态映射

### 3.1 分数到状态的映射

```typescript
export enum InspirationStatus {
  SEED = 'seed',      // 种子 - 刚刚开始
  SPROUT = 'sprout',  // 萌芽 - 初步发展
  GROW = 'grow',      // 生长 - 逐渐成长
  BUD = 'bud',        // 含苞 - 接近成熟
  BLOOM = 'bloom',    // 绽放 - 完整成熟
}
```

### 3.2 状态阈值配置

| 状态 | 分数区间 | 描述 | 液体高度 |
|------|---------|------|----------|
| SEED（种子） | 0-20分 | 只有一个想法种子 | 0-20% |
| SPROUT（萌芽） | 20-40分 | 开始有初步内容 | 20-40% |
| GROW（生长） | 40-60分 | 内容逐渐丰富 | 40-60% |
| BUD（含苞） | 60-80分 | 比较完整，接近成熟 | 60-80% |
| BLOOM（绽放） | 80-100分 | 灵感已成熟完整 | 80-100% |

**阈值配置**：
```typescript
const DEFAULT_STATUS_THRESHOLDS = {
  seed: 20,    // 0-20 → SEED
  sprout: 40,  // 20-40 → SPROUT
  grow: 60,    // 40-60 → GROW
  bud: 80,     // 60-80 → BUD
  bloom: 80,   // 80-100 → BLOOM
}
```

## 四、评分算法

### 4.1 评分流程

```text
输入灵感 → 评估4个维度 → 计算加权总分 → 映射到状态 → 生成建议
    ↓                ↓              ↓            ↓
每个维度独立计算  加权求和  按阈值判断状态  低分维度→建议
```

### 4.2 计算公式

```
总分 = Σ(维度分数 × 维度权重)
其中：
- 维度分数：0-100
- 维度权重：0-1（总和 = 1）
- 总分：0-100（保留两位小数）
```

**示例计算**：

假设某个灵感：
- 清晰度：70分 × 0.25 = 17.5
- 丰富度：85分 × 0.30 = 25.5
- 可行性：60分 × 0.25 = 15.0
- 独特性：45分 × 0.20 = 9.0
- **总分** = 17.5 + 25.5 + 15.0 + 9.0 = **67.0分**

**状态映射**：67分 → BUD（含苞）

### 4.3 建议生成逻辑

对于得分 < 60 的维度，生成改进建议：

| 维度 | 建议内容 |
|------|---------|
| 清晰度低 | "建议：添加更清晰的名称和详细描述" |
| 丰富度低 | "建议：添加图片、语音或参考链接" |
| 可行性低 | "建议：进行头脑风暴，思考执行步骤" |
| 独特性低 | "建议：尝试与其他灵感碰撞" |

## 五、接口定义

### 5.1 评分结果结构

```typescript
interface ScoringResult {
  totalScore: number;          // 总分 (0-100)
  dimensionScores: {           // 各维度得分
    dimension: ScoringDimension;
    name: string;
    score: number;
    weight: number;
    weightedScore: number;
    reason: string;            // 得分原因
  }[];
  status: InspirationStatus;   // 映射后的状态
  evaluatedAt: number;         // 评分时间戳
  suggestions: string[];       // 改进建议列表
}
```

### 5.2 核心API

```typescript
// 1. 完整评估（返回所有信息）
const result = evaluator.evaluate(inspiration);
// { totalScore: 67, dimensionScores: [...], status: 'bud', ... }

// 2. 快速评估（仅返回总分和状态）
const quick = evaluator.quickEvaluate(inspiration);
// { score: 67, status: 'bud' }

// 3. 便捷函数（使用默认配置）
const result = evaluateInspiration(inspiration);
const quick = quickEvaluateInspiration(inspiration);
```

## 六、使用场景

### 6.1 灵感创建后
新灵感创建完成后自动评分，设置初始状态

### 6.2 内容更新后
用户添加图片、文字、头脑风暴卡片后重新评估

### 6.3 灵感碰撞后
多个灵感碰撞生成新灵感时进行评估

### 6.4 用户主动刷新
用户可手动触发重新评估（主要用于调试）

## 七、扩展和优化建议

### 7.1 可增强功能

1. **LLM辅助评分**
   - 本地评分 + LLM确认
   - 更智能的内容理解
   - 生成个性化建议

2. **时间衰减**
   - 长时间未更新的灵感轻微降分
   - 鼓励持续维护和更新

3. **进度可视化**
   - 四维雷达图展示各维度得分
   - 历史评分趋势图表

4. **自定义权重**
   - 允许用户调整各维度权重
   - 不同类型灵感可使用不同权重

5. **评分缓存**
   - 避免重复计算
   - 内容未变更时复用上次结果
